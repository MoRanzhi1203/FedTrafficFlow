# 速度等级到密度与流量计算方案（修订版）

## 1. 方案目的

本文档用于统一项目中“速度等级 - 自由流速度 - 临界密度 - 阻塞密度 - 超速处理 - 密度与流量计算”的参数设定与程序实现规则，供论文撰写、实验配置和脚本开发共用。

本方案基于 `data/analysis/speed_histograms_by_class_p995.csv` 中的 P99.5 结果，对不同速度等级进行标准化归档。核心原则是：

```text
P99.5 只用于辅助校准速度档位；
Greenshields 模型中的自由流速度 v_f 使用标准化道路速度档位；
临界密度 k_c 与阻塞密度 k_j 必须区分；
速度到密度、流量的转换遵循 Greenshields 速度-密度基本关系。
```

为避免符号混淆，本文档统一采用交通工程中常用符号：

| 符号 | 含义 | 单位 |
| --- | --- | --- |
| `v_i(t)` | 路段 `i` 在时刻 `t` 的观测平均速度 | km/h |
| `v_{f,i}` | 路段 `i` 的自由流速度 | km/h |
| `v_i^{used}(t)` | 用于计算的截断速度 | km/h |
| `k_{c,i}^{lane}` | 每车道临界密度 | veh/km/lane |
| `k_{j,i}^{lane}` | 每车道阻塞密度 | veh/km/lane |
| `k_{j,i}^{road}` | 路段总阻塞密度 | veh/km |
| `k_i(t)` | 路段总交通密度 | veh/km |
| `q_i(t)` | 路段小时交通流率 | veh/h |
| `Q_i^{15min}(t)` | 15 分钟交通量 | veh/15min |
| `n_i` | 车道数 | lane |
| `L_i` | 路段长度 | km |

## 2. 核心结论

项目当前建议使用以下三档标准自由流速度：

```text
速度等级 2、3       -> 100 km/h
速度等级 4          -> 80 km/h
速度等级 5、6、7、8 -> 60 km/h
```

对应参数如下：

| 速度等级 | P99.5 速度 | 建议自由流速度 `v_f` | 每车道临界密度 `k_c^{lane}` | 每车道阻塞密度 `k_j^{lane}=2k_c^{lane}` | 说明 |
| ---: | ---: | ---: | ---: | ---: | --- |
| 2 | 97.18 | 100 | 42 | 84 | 接近 100 km/h，适合高等级城市快速路 |
| 3 | 94.28 | 100 | 42 | 84 | 接近 100 km/h，适合高等级城市快速路 |
| 4 | 81.74 | 80 | 46 | 92 | 与 80 km/h 档最接近 |
| 5 | 63.61 | 60 | 50 | 100 | 接近 60 km/h，轻微超过部分按超速或异常处理 |
| 6 | 59.70 | 60 | 50 | 100 | 与 60 km/h 档高度一致 |
| 7 | 52.90 | 60 | 50 | 100 | 可视为低等级道路或低速运行路段 |
| 8 | 60.18 | 60 | 50 | 100 | 样本量较小，不单独成档 |

因此，当前数据更支持 `100 / 80 / 60 km/h` 三档，而不建议直接采用 `97.175`、`94.279`、`81.738` 这类统计小数值作为模型参数。

需要特别说明的是：原方案中的 `42 / 46 / 50 veh/km/lane` 应解释为每车道**临界密度** `k_c^{lane}`，而不是 Greenshields 公式中的阻塞密度 `k_j`。在标准 Greenshields 模型中，最大流量发生在：

```text
k_c = k_j / 2
```

因此，程序实现中应先由临界密度换算阻塞密度：

```text
k_j^{lane} = 2 * k_c^{lane}
```

再进入密度计算。

## 3. 四环、五环与 120 km/h 的处理原则

本项目不建议把四环、五环统一按 `120 km/h` 处理。

更合理的工程解释是：

- 城市快速路中的高等级路段优先归入 `100 km/h` 档。
- 一般主干路或限速较低的快速路归入 `80 km/h` 档。
- 仅当道路明确属于高速公路体系或城市外高速连接线时，才考虑单独使用 `120 km/h` 档。

结合当前 P99.5 结果，速度等级 2 和 3 的高分位速度分别为 `97.18 km/h` 和 `94.28 km/h`，更支持把高等级道路自由流速度设为 `100 km/h`，而非 `120 km/h`。

---

## 4. 为什么不直接用 P99.5 作为自由流速度

不建议直接令：

```text
v_f = P99.5
```

原因如下：

- `Greenshields` 模型中的 `v_f` 应具有道路物理意义，应对应设计速度、自由流速度或标准速度档位，而不是纯统计分位数。
- `P99.5` 受样本分布影响较大，低样本等级可能出现不稳定值。
- 直接使用分位数会导致参数碎片化，不利于论文叙述，也不利于和标准速度档位建立对应关系。
- 标准化档位便于后续不同路段、不同客户端和不同实验批次之间保持参数一致。

因此，本项目采用：

```text
P99.5 用于校准速度档位；
密度计算使用标准化后的 100、80、60 km/h。
```

## 5. Greenshields 密度计算公式

对每条道路 `i`，先根据其速度等级确定：

- 自由流速度 `v_{f,i}`
- 每车道临界密度 `k_{c,i}^{lane}`
- 每车道阻塞密度 `k_{j,i}^{lane}`

两者关系为：

```text
k_{j,i}^{lane} = 2 * k_{c,i}^{lane}
```

再结合车道数 `n_i` 得到路段总阻塞密度：

```text
k_{j,i}^{road} = k_{j,i}^{lane} * n_i
```

对任意 15 分钟时段，定义路段观测平均速度为 `v_i(t)`。为避免超速或异常值导致负密度，程序中采用截断后的速度：

```text
v_i^{used}(t) = min(v_i(t), v_{f,i})
```

Greenshields 速度-密度关系为：

```text
v_i^{used}(t) = v_{f,i} * (1 - k_i(t) / k_{j,i}^{road})
```

反解得到路段总密度：

```text
k_i(t) = k_{j,i}^{road} * (1 - v_i^{used}(t) / v_{f,i})
```

并进一步保证非负：

```text
k_i(t) = max(0, k_i(t))
```

说明：

```text
这里的 k_i(t) 已经是路段总密度，包含车道数影响。
因此后续计算流量时不再额外乘以车道数 n_i。
```

## 6. 超速与异常高速观测处理

本项目对超速采用“两层处理”。

### 6.1 第一层：物理截断

若观测速度超过该等级自由流速度：

```text
v_i(t) > v_{f,i}
```

则计算密度时使用：

```text
v_i^{used}(t) = v_{f,i}
```

此时：

```text
k_i(t) = 0
```

该处理用于避免出现负密度，也符合“速度高于自由流速度时视为极低密度或近似空载”的工程解释。

### 6.2 第二层：异常标记

若观测速度超过该速度等级对应的 P99.5：

```text
v_i(t) > P99.5_class
```

则可额外标记：

```text
high_speed_outlier = 1
```

该标记用于数据质量分析，不强制删除样本。密度计算仍然只基于 `v_f` 截断规则。

## 7. 派生量计算

若已得到路段总密度 `k_i(t)`，并记道路长度为 `L_i`，其中长度单位为 `km`，则：

### 7.1 路段车辆数

```text
N_i(t) = k_i(t) * L_i
```

其中 `N_i(t)` 的单位为 `veh`。

### 7.2 小时交通流率

项目中建议使用截断后的速度 `v_i^{used}(t)` 计算派生流量，以保持和密度定义一致：

```text
q_i(t) = v_i^{used}(t) * k_i(t)
```

其中：

```text
v_i^{used}(t): km/h
k_i(t): veh/km
q_i(t): veh/h
```

注意：

```text
由于 k_i(t) 已经是路段总密度，q_i(t) = v_i^{used}(t) * k_i(t) 中不再额外乘以 n_i。
```

### 7.3 15 分钟交通量

若时间粒度为 15 分钟，则：

```text
Q_i^{15min}(t) = q_i(t) / 4
```

### 7.4 理论最大流量

在 Greenshields 模型下，最大流量发生于：

```text
k_i(t) = k_{j,i}^{road} / 2
```

此时：

```text
q_{i,max} = v_{f,i} * k_{j,i}^{road} / 4
```

该值可用于流量归一化和模型输入特征构造。

## 8. 归一化状态变量与后续模型衔接

为便于后续马尔可夫状态转移、图基空间建模和神经网络预测，建议在密度和流量计算后额外构造以下物理特征。

### 8.1 归一化密度

```text
rho_i(t) = k_i(t) / k_{j,i}^{road}
```

其中：

```text
rho_i(t) in [0, 1]
```

### 8.2 归一化流量

```text
q_tilde_i(t) = q_i(t) / q_{i,max}
```

其中：

```text
q_{i,max} = v_{f,i} * k_{j,i}^{road} / 4
```

### 8.3 离散交通状态

可根据归一化密度 `rho_i(t)` 将连续交通状态离散化：

```text
S_i(t) = 0, if 0.0 <= rho_i(t) < 0.3   # 自由流
S_i(t) = 1, if 0.3 <= rho_i(t) < 0.5   # 稳定流
S_i(t) = 2, if 0.5 <= rho_i(t) < 0.7   # 接近饱和
S_i(t) = 3, if 0.7 <= rho_i(t) <= 1.0  # 拥堵流
```

该状态变量不参与速度到流量的物理转换本身，而是作为后续马尔可夫模型、图基预测方法或神经网络输入的增强特征。

### 8.4 推荐输出物理特征向量

推荐为每条路段、每个时间片输出：

```text
X_i^{phy}(t) = [
    v_i(t),
    v_i^{used}(t),
    k_i(t),
    q_i(t),
    Q_i^{15min}(t),
    rho_i(t),
    q_tilde_i(t),
    S_i(t),
    high_speed_outlier
]
```

## 9. 与马尔可夫模型和图基预测方法的关系

本方案中的 Greenshields 模型只负责完成：

```text
速度 v -> 密度 k -> 流量 q
```

马尔可夫模型和图基预测方法不应替代该物理转换过程，而应放在物理特征构造之后，用于增强预测模型的时空解释能力。

### 9.1 马尔可夫模型的使用位置

在得到离散交通状态 `S_i(t)` 后，可构造状态转移概率：

```text
P_i(a, b) = P(S_i(t+1) = b | S_i(t) = a)
```

若考虑邻居路段影响，可扩展为图马尔可夫形式：

```text
P_i,t(a, b) = P(S_i(t+1) = b | S_i(t) = a, S_j(t), j in N(i))
```

马尔可夫结果可作为神经网络的额外输入特征，例如：

```text
pi_i(t+1) = [
    P(S_i(t+1)=0),
    P(S_i(t+1)=1),
    P(S_i(t+1)=2),
    P(S_i(t+1)=3)
]
```

### 9.2 图基预测方法的使用位置

图基预测方法主要用于神经网络空间建模，而不是用于速度到流量转换。

道路网络可表示为：

```text
G = (V, E, A)
```

其中 `A` 为邻接矩阵，可由道路拓扑、空间距离或历史流量相似性构造。图卷积空间特征可写为：

```text
H_t = sigma(D_tilde^{-1/2} A_tilde D_tilde^{-1/2} X_t^{phy} W)
```

因此，推荐总体流程为：

```text
原始速度数据
-> Greenshields 物理特征构造
-> 马尔可夫状态转移特征 / 图结构空间特征
-> CNN / GCN / BiLSTM / Attention 等神经网络预测模型
-> 联邦聚合与多区域协同预测
```

## 10. 程序实现规则

项目脚本 `analysis_scripts/compute_greenshields_density.py` 建议按以下规则实现或更新：

- 速度等级 `2`、`3` 映射到 `100 km/h` 和 `42 veh/km/lane` 的临界密度 `k_c^{lane}`
- 速度等级 `4` 映射到 `80 km/h` 和 `46 veh/km/lane` 的临界密度 `k_c^{lane}`
- 速度等级 `5`、`6`、`7`、`8` 映射到 `60 km/h` 和 `50 veh/km/lane` 的临界密度 `k_c^{lane}`
- 每车道阻塞密度按 `k_j^{lane} = 2 * k_c^{lane}` 计算
- 路段总阻塞密度按 `k_j^{road} = k_j^{lane} * n_i` 计算
- 超过自由流速度的观测在密度计算时截断
- 超过 P99.5 的观测单独标记为异常高速
- 输出截断速度、每车道临界密度、每车道阻塞密度、路段总阻塞密度、路段总密度、路段车辆数、小时交通流率、15 分钟交通量、归一化密度、归一化流量、离散交通状态和异常高速标记等字段

推荐输出字段包括：

```text
speedclass
v_obs
v_f
v_used
p995_class
high_speed_outlier
lane_num
k_c_lane
k_j_lane
k_j_road
density_k
segment_length_km
vehicle_count_N
flow_q_hour
flow_Q_15min
rho
q_max
q_tilde
traffic_state_S
```

## 11. 适用说明

- 本方案适用于当前仓库内带有 `速度等级`、`车道数`、`长度`、`平均速度` 的分块交通速度结果。
- 本方案默认 `长度` 字段单位为 `km`。若后续接入的数据源以 `m` 为单位，需要先转换为 `km` 再计算车辆数。
- 若未来加入高速公路数据并有充分依据，可扩展 `120 km/h` 档；但不建议在当前城市路网样本上默认启用。
- 若后续采用外部容量标准进行校准，应将“容量参数”和 “Greenshields 密度转换参数”分开管理，避免把临界密度、阻塞密度和道路容量混用。
- 本方案中的马尔可夫和图基预测内容仅用于说明后续建模衔接关系，不改变速度到密度、流量的物理转换主流程。
