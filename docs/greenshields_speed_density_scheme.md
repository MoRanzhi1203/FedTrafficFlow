# 速度等级到密度计算方案

## 1. 方案目的

本文档用于统一项目中“速度等级 - 自由流速度 - 临界密度 - 超速处理 - 密度计算”的参数设定与程序实现规则，供论文撰写、实验配置和脚本开发共用。

本方案基于 `data/analysis/speed_histograms_by_class_p995.csv` 中的 P99.5 结果，对不同速度等级进行标准化归档。核心原则是：P99.5 只用于辅助校准速度档位，不直接作为 Greenshields 模型中的自由流速度。

## 2. 核心结论

项目当前建议使用以下三档标准自由流速度：

```text
速度等级 2、3 -> 100 km/h
速度等级 4    -> 80 km/h
速度等级 5、6、7、8 -> 60 km/h
```

对应参数如下：

| 速度等级 | P99.5 速度 | 建议自由流速度 `v_f` | 每车道临界密度 `d_j^{lane}` | 说明 |
| ---: | ---: | ---: | ---: | --- |
| 2 | 97.18 | 100 | 42 | 接近 100 km/h，适合高等级城市快速路 |
| 3 | 94.28 | 100 | 42 | 接近 100 km/h，适合高等级城市快速路 |
| 4 | 81.74 | 80 | 46 | 与 80 km/h 档最接近 |
| 5 | 63.61 | 60 | 50 | 接近 60 km/h，轻微超过部分按超速或异常处理 |
| 6 | 59.70 | 60 | 50 | 与 60 km/h 档高度一致 |
| 7 | 52.90 | 60 | 50 | 可视为低等级道路或低速运行路段 |
| 8 | 60.18 | 60 | 50 | 样本量较小，不单独成档 |

因此，当前数据更支持 `100 / 80 / 60 km/h` 三档，而不建议直接采用 `97.175`、`94.279`、`81.738` 这类统计小数值作为模型参数。

## 3. 四环、五环与 120 km/h 的处理原则

本项目不建议把四环、五环统一按 `120 km/h` 处理。

更合理的工程解释是：

- 城市快速路中的高等级路段优先归入 `100 km/h` 档。
- 一般主干路或限速较低的快速路归入 `80 km/h` 档。
- 仅当道路明确属于高速公路体系或城市外高速连接线时，才考虑单独使用 `120 km/h` 档。

结合当前 P99.5 结果，速度等级 2 和 3 的高分位速度分别为 `97.18 km/h` 和 `94.28 km/h`，更支持把高等级道路自由流速度设为 `100 km/h`，而非 `120 km/h`。

## 4. 为什么不直接用 P99.5 作为自由流速度

不建议直接令：

```text
v_f = P99.5
```

原因如下：

- `Greenshields` 模型中的 `v_f` 应具有道路物理意义，应对应设计速度或自由流速度档位，而不是纯统计分位数。
- `P99.5` 受样本分布影响较大，低样本等级可能出现不稳定值。
- 直接使用分位数会导致参数碎片化，不利于论文叙述，也不利于和标准速度档位建立对应关系。

因此，本项目采用：

```text
P99.5 用于校准档位；
密度计算使用标准化后的 100、80、60 km/h。
```

## 5. 密度计算公式

对每条道路 `i`，先根据其速度等级确定：

- 自由流速度 `v_{f,i}`
- 每车道临界密度 `d_{j,i}^{lane}`

再结合车道数 `n_i` 得到路段总临界密度：

```text
d_{j,i}^{road} = d_{j,i}^{lane} * n_i
```

对任意 15 分钟时段，定义该路段观测平均速度为 `v_i(t)`，则 Greenshields 密度计算为：

```text
d_i(t) = d_{j,i}^{road} * (1 - v_i(t) / v_{f,i})
```

为避免超速或异常值导致负密度，程序中采用截断后的速度：

```text
v_i^{used}(t) = min(v_i(t), v_{f,i})
```

实际计算式写为：

```text
d_i(t) = d_{j,i}^{road} * (1 - v_i^{used}(t) / v_{f,i})
```

并进一步保证非负：

```text
d_i(t) = max(0, d_i(t))
```

## 6. 超速与异常高速观测处理

本项目对超速采用“两层处理”。

### 6.1 第一层：物理截断

若观测速度超过该等级自由流速度：

```text
v_i(t) > v_{f,i}
```

则计算密度时使用：

```text
v_i^{used}(t) = v_{f,i}
```

此时密度记为 `0`，从而避免出现负密度。

### 6.2 第二层：异常标记

若观测速度超过该速度等级对应的 P99.5：

```text
v_i(t) > P99.5_class
```

则可额外标记：

```text
high_speed_outlier = 1
```

该标记用于数据质量分析，不强制删除样本。密度计算仍然只基于 `v_f` 截断。

## 7. 派生量计算

若已得到密度 `d_i(t)`，并记道路长度为 `L_i`，其中长度单位为 `km`，则：

### 7.1 路段车辆数

```text
N_i(t) = d_i(t) * L_i
```

### 7.2 流量

项目中建议使用截断后的速度 `v_i^{used}(t)` 计算派生流量，以保持和密度定义一致：

```text
q_i(t) = v_i^{used}(t) * d_i(t)
```

其中 `q_i(t)` 的单位为 `veh/h`。

15 分钟流量可写为：

```text
Q_i^{15min}(t) = q_i(t) / 4
```

## 8. 程序实现规则

项目脚本 `analysis_scripts/compute_greenshields_density.py` 按以下规则实现：

- 速度等级 `2`、`3` 映射到 `100 km/h` 和 `42 veh/km/lane`
- 速度等级 `4` 映射到 `80 km/h` 和 `46 veh/km/lane`
- 速度等级 `5`、`6`、`7`、`8` 映射到 `60 km/h` 和 `50 veh/km/lane`
- 超过自由流速度的观测在密度计算时截断
- 超过 P99.5 的观测单独标记为异常高速
- 输出路段临界密度、截断速度、密度、车辆数和流量等字段

## 9. 适用说明

- 本方案适用于当前仓库内带有 `速度等级`、`车道数`、`长度`、`平均速度` 的分块交通速度结果。
- 本方案默认 `长度` 字段单位为 `km`。若后续接入的数据源以 `m` 为单位，需要先转换为 `km` 再计算车辆数。
- 若未来加入高速公路数据并有充分依据，可扩展 `120 km/h` 档；但不建议在当前城市路网样本上默认启用。
