# analysis_scripts 分类规则

## 目录说明

- `audits`: 审计、盘点、质量核查类脚本，用于输出检查报告、库存清单或一致性结论。
- `real_data_pipeline`: 真实交通数据的核心派生处理脚本，负责统计汇总、指标计算、节点流量构建与曲线拟合等主流程。
- `real_data_analysis`: 基于真实数据处理结果开展对比分析、可视化和模型/阶数比较的脚本。
- `missingness_pipeline_core`: 缺失机制构造、缺失数据生成、补全实验执行等核心 pipeline 脚本。
- `missingness_result_analysis`: 针对缺失实验结果的分析、分布检查与可视化脚本。
- `missingness_support`: 缺失实验的路径配置、结果整理、布局迁移、修复与维护辅助脚本。

## 归类标准

- 若脚本主要负责生成中间结果/正式结果数据，归入对应 `pipeline` 类目录。
- 若脚本主要负责绘图、对比、统计解释、结果检查，归入 `analysis` 类目录。
- 若脚本主要负责修复目录、合并结果、清理产物、共享路径配置，归入 `support` 类目录。
- 若脚本主要负责全仓扫描、审计、质量清单输出，归入 `audits` 目录。

## 兼容策略

- 所有原始脚本已移动到上述子目录。
- 为兼容现有 `analysis_scripts.<module>` 导入方式及根层脚本调用方式，在 `analysis_scripts` 根目录保留了同名硬链接入口。
- 后续新增脚本建议优先放入对应子目录；若需要兼容旧路径调用，可按同样方式在根目录创建同名兼容入口。
