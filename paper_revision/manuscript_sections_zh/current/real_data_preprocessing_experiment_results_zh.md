# 真实数据预处理与数据质量分析

## 1. 真实交通数据来源与基本结构

本文真实数据实验部分基于实际交通观测数据展开，当前项目中的原始输入文件包括 `link_gps.v2`、`road_network_sub-dataset.v2` 与 `traffic_speed_sub-dataset.v2`。其中，前两类文件分别提供路段空间坐标信息与路网属性、节点连接关系等静态结构信息，后一类文件提供按时间组织的交通速度观测。由此可以看出，当前真实数据链路以“路段静态属性 + 路段速度观测”的组合形式构成基础输入，再逐步转换为节点级交通流时序结果。

从结构上看，`link_gps_processed.csv` 与 `rnsd_processed.csv` 在清洗后均保留 45,148 条路段记录，说明路段坐标表与路网属性表在主键尺度上保持了一致的基础规模。进一步地，节点完整性检查结果显示，拓扑节点数与观测节点数均为 42,031，表明现有真实数据链路已经能够将路段级数据稳定映射到节点级表示。

需要说明的是，当前项目文件中未发现明确的数据集名称标识，正式论文中仍需补充真实数据来源、采集区域和引用信息。

## 2. 数据清洗与质量控制

预处理流程主要包括路段坐标清洗、路网属性清洗、速度观测与静态属性合并、流量派生以及节点级完整性检查五个环节。首先，在路段坐标处理阶段，代码对 `link_gps.v2` 执行去重、关键字段空值删除和类型统一，并输出标准化坐标表。其次，在路网属性处理阶段，项目对 `road_network_sub-dataset.v2` 完成英文字段到中文字段的映射，删除重复路段与关键字段缺失记录，并基于长度与方向推导起始节点和结束节点对应的空间坐标。

在动态观测处理阶段，原始速度观测与清洗后的路段静态属性进行关联，并按照时间段区间写出结构化分片结果。随后，项目基于速度与参数表进一步计算密度和流量相关派生指标，再将路段流量聚合为节点进入流量、节点离开流量与节点综合流量。为保证物理含义的一致性，现有实现会删除负车流量记录，并在进入流量与离开流量外连接聚合后，将缺失值补为 0。

由数据质量统计可知，节点流量完整性检查覆盖了 61 个日分片，且每日均包含 96 个时间段；总观测节点数与拓扑节点数一致，均为 42,031；总原始记录数、总期望记录数和总唯一观测对数均为 246,133,536；缺失记录、重复记录、空值、NaN 与负值计数均为 0。上述结果说明，当前项目已经对节点级真实数据完成较严格的结构一致性与完整性控制。

需要说明的是，当前代码中暂未发现明确的异常值裁剪逻辑，后续仍需结合数据说明或补充代码确认是否存在更细粒度的异常值平滑、截断或插值机制。

## 3. 时间序列样本构造

现有真实数据处理链路已经形成按时间段组织的节点级时序数据，并进一步构造了节点日内平均曲线与傅里叶拟合结果。从实现方式看，时间组织主要依赖整数型 `时间段` 索引，节点流量文件按 `[时间段, 节点ID]` 排序，并以每日 96 个时间段为基本检查单元，从而保证时序结构的连续性与一致性。

然而，当前扫描未定位到明确的面向预测任务的输入窗口长度、预测步长与滑动窗口样本构造逻辑。换言之，当前项目能够确认的是真实数据已经被整理为可连续分析的节点级时间序列，但针对联邦交通流预测模型的监督学习样本生成过程仍需在后续代码审查中进一步确认。

因此，在正式论文表述中，可将现阶段真实数据链路概括为“先形成时序一致的节点级流量分片，再开展日内曲线分析与拟合”，而不宜直接写入尚未被证据支持的窗口长度或预测步长设定。

## 4. 空间节点与图结构构造

当前项目中，空间节点信息主要来源于 `rnsd_processed.csv` 中的 `起始节点ID` 与 `结束节点ID`。基于该路网拓扑映射，项目将路段级流量分别聚合到节点进入方向与离开方向，并进一步生成节点综合车流量。这表明真实数据链路已经建立了从“路段观测”到“节点观测”的空间映射关系。

从统计结果看，拓扑节点数与观测节点数均为 42,031，且不存在“观测中出现但拓扑中不存在”或“拓扑中存在但观测中缺失”的节点，这为后续真实数据分析提供了较好的空间一致性基础。与此同时，节点流量完整性检查并未依赖额外的空间填补过程，而是直接在原始拓扑节点集合上验证覆盖情况。

需要说明的是，当前项目中未发现与真实数据对应的独立邻接矩阵输出文件，也未发现基于距离阈值、相关性阈值或动态图规则显式构造真实数据图结构的单独脚本。因此，后续仍需确认真实数据实验是否在预测模型阶段显式使用空间邻接信息，或者仅以节点聚合结果作为输入。

## 5. 联邦客户端划分

从当前真实数据预处理链路来看，项目已经形成了能够支撑节点级时空分析的数据底座，但尚未扫描到与真实数据实验直接对应的联邦客户端划分实现。现有代码中未发现基于区域、节点集合、传感器集合或时间片划分客户端的明确规则，也未发现客户端数量、客户端样本量或客户端级输出文件。

因此，在正式论文中不宜直接写入具体客户端数量或划分策略。较为稳妥的表述是：当前扫描未发现真实数据实验中明确的客户端划分规则，需要后续补充说明，以保证联邦实验设计的可复核性。

## 6. 归一化与数据集划分

根据现有扫描结果，当前项目中暂未发现针对真实数据预测样本的 `MinMaxScaler`、`StandardScaler` 或类似归一化器拟合与保存逻辑，也未发现真实数据训练集、验证集、测试集的显式划分脚本与比例说明。现有代码中出现的归一化操作主要用于日期类型曲线构造与聚类分析，其目的在于消除曲线尺度差异并突出形态特征，不应直接等同于预测任务输入归一化。

基于这一结果，正式论文中宜明确区分“曲线分析阶段的形态归一化”和“预测模型阶段的数据预处理归一化”。前者在当前项目中已有实现，后者则仍缺少直接证据。此外，当前项目中暂未发现 scaler 参数保存文件，后续需确认归一化是否具备可复现性。

同样地，训练/验证/测试划分比例目前也暂未确认，因此正式稿中不应写入具体划分数值，而应保留为后续待补充信息。

## 7. 预处理结果统计

从现有审查结果可以提炼出以下真实数据预处理统计信息。

- 原始数据文件数量为 3 个，分别对应路段坐标、路网属性和速度观测。
- 其中，`traffic_speed_sub-dataset.v2` 文件体积约为 8.29 GB，说明动态观测规模较大；由于文件体积较大，本次审查仅完成结构与样本级质量统计，未对其全量行数进行离线计数。
- `link_gps_processed.csv` 为 45,148 行、3 列，`rnsd_processed.csv` 为 45,148 行、12 列，说明静态路段表已完成标准化整理。
- 节点流量完整性检查覆盖 61 个日分片、42,031 个节点以及每日 96 个时段，总唯一节点-时间观测对数为 246,133,536。
- 完整性汇总表显示，缺失记录、重复记录、空值、NaN 与负值均为 0，说明节点级预处理结果在结构一致性方面通过了检查。
- 按日统计的节点平均流量约在 2376.59 至 2459.67 之间，单日最大流量约在 10260.16 至 10477.75 之间；同时，每日零流量记录约在 18,406 至 35,994 之间，表明零流量状态客观存在，但未造成缺失或非法记录问题。
- 当前项目中暂未确认预处理后预测样本量、训练/验证/测试样本量以及客户端样本量，因此这些统计仍需后续补充。

总体而言，当前预处理结果已经能够证明真实数据链路在空间映射、时序覆盖与完整性控制方面具备较好的稳定性，但对“可直接进入联邦预测训练”的样本化结果仍缺少证据闭环。

## 8. 小结

综上，当前真实数据预处理流程已经形成了从原始路段与路网信息读取、到速度观测关联、再到节点级流量构造与完整性检查的连续链路，为后续真实数据实验提供了统一的输入基础和数据质量保障。尤其是节点级结果在 61 天、42,031 个节点和 246,133,536 条观测对上的零缺失、零重复、零非法值统计，说明现有数据整理工作已经达到较高的一致性水平。

需要说明的是，真实数据来源说明、预测样本窗口构造、客户端划分、训练/验证/测试划分以及预测阶段归一化与参数保存等内容在当前项目中仍缺少明确证据。因而，在最终论文定稿前，仍需进一步补充这些环节，以增强真实数据实验部分的完整性与可复现性。
